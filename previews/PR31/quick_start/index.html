Quick start

Suppose we want to sum a vector, which is the result of multiple broadcast expressions: y = x .+ x, z = 2 .* y, and, finally, sum(z). We might first write this as:

using Base.Broadcast: materialize
using LazyBroadcast: lazy_broadcast
using BenchmarkTools

function foo(x)
   y = x .+ x
   z = 2 .* y
   sum(z)
end;

print(@btime foo(v) setup=(v=rand(10)))
  50.541 ns (4 allocations: 288 bytes)
16.419734991214785

As you can see, there are heap allocations, which are impacting performance. Now, let's use LazyBroadcast.jl's lazy_broadcast to eliminate the intermediate allocations:

function foo_lazy(x)
   # use lazy_broadcast to avoid intermediate allocations
   y = lazy_broadcast.(x .+ x)
   z = lazy_broadcast.(2 .* y)
   sum(z)
end;

print(@btime foo_lazy(v) setup=(v=rand(10)))
  10.189 ns (0 allocations: 0 bytes)
18.6303794166443

On my computer, the benchmarks take 43.433 ns and 5.917 ns, respectively. So, using LazyBroadcast results in a 7x speedup. This demonstrates the power of using lazy_broadcast, which has allowed us to avoid temporary allocations.

If you're interested in the implementation, check out the Internals of how lazy_broadcast works section.
